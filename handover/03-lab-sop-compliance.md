Our code will be organized in a clear and consistent manner, following standard professional guidelines. This includes:

*   A well-structured directory hierarchy with clear naming conventions

*   Meaningful commit messages and frequent commits

We will use clear and concise commenting to explain the purpose and functionality of each code snippet.

*   Clear variable names and comments for complex logic
*   Use of documentation tools like doxygen or sphinx
*   Consistent formatting for comments and docstrings

We will implement thorough testing and quality assurance procedures to ensure the stability and reliability of our codebase.

*   Unit tests and integration tests for individual components
*   Continuous Integration/Continuous Deployment (CI/CD) pipelines
*   Code review and peer feedback

We will adhere to industry-standard best practices and guidelines for coding, testing, and documentation.

*   Follow PEP8 or Google Style for Python/C++/Java code
*   Use version control best practices (e.g., Git)

Our lab SOP compliance will be ensured through regular audits and checks.

*   Regularly review and update our SOP document
*   Implement checklists and workflows for routine tasks

### Lab Formatting Guidelines

- When interpreting, generating, or modifying manifest files related to apps and agents for Microsoft 365 or Microsoft 365 Copilot, try to get schema version from manifest file, then invoke \*\*get\_schema\*\* tool if available.
- When answering of what or how-to questions for building apps and agents for Microsoft 365 or Microsoft 365 Copilot, invoke \*\*get\_knowledge\*\* tool if available.
- Before deciding to make changes to Teams apps, apps or agents for Microsoft 365 or Microsoft 365 Copilot, invoke \*\*get\_knowledge\*\* tool to get Instructions if available.
- When troubleshooting issues, invoke \*\*troubleshoot\*\* tool if available.
- Before generating or modifying code or configuration files for apps and agents for Microsoft 365 or Microsoft 365 Copilot, invoke \*\*get\_code\_snippets\*\* with API name, configuration file name, or code comments every time you need to generate or modify code or configuration files for apps and agents for Microsoft 365 or Microsoft 365 Copilot.
